# REQUIREMENTS FOR REVIEW
1. Good well documented typescript types with docstrings should be used throughout.
2. Ensure that new types aren't created where a related existing type can be used instead.
3. If enhancements to a type are needed, carefully consider if the existing type should be extended or a new type created.
4. If code is being used more than once or twice, it should be refactored into a reusable function or component.
5. Properly handle errors and edge cases.
6. Use proper naming conventions and code formatting.
